# AGENTS.md — ease-design

Guidance for Codex CLI and other `AGENTS.md`-aware agents working **in this repository**
to develop ease-design itself.

> Runtime-neutral config stub. The real per-runtime adapters that teach a host CLI how to
> *use* ease-design as a design tool are generated by `ui init` (Phase 6).

## Project

ease-design is a multi-runtime design-cli — see `README.md`. It ships two sources of
truth:

- **`knowledge/`** — plain-Markdown knowledge core, read directly by the host model.
- **`ui` binary** (`src/cli.ts`) — deterministic Node CLI for all non-LLM work.

## Commands

| Task | Command |
|---|---|
| Type-check | `npm run typecheck` |
| Lint | `npm run lint` |
| Build the `ui` binary | `npm run build` |
| Test | `npm test` |

Run all four before committing; CI enforces them. Requires Node.js >= 20.

## Rules

- The `ui` binary stays deterministic — pure transforms, no network, no model calls.
- Do not duplicate `knowledge/` content into code.
- Decisions in `plans/ease-design/brainstorm.md` §6 are locked.
