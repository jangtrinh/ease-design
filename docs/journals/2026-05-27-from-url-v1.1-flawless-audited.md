# `/ui:from-url` v1.1 — making the output flawless via a deterministic audit gate

**Date**: 2026-05-27
**Severity**: Low (additive feature; closes systemic gaps in v1.x)
**Component**: `src/commands/designmd-*.ts`, `src/core/designmd-*.ts`, `src/core/url-slug.ts`, `templates/workflows/from-url.md`, `templates/skills/designmd-emit.md`, `knowledge/designmd-format.md`, four dogfood folders under `plans/260527-from-url-v1.1-flawless-audited/artifacts/`
**Status**: Shipped — v1.1 feature complete, dogfooded against 4 URL classes with PASS/WARN audit verdicts

## Context

v1.x of `/ui:from-url` shipped this morning. Within hours, dogfooding
it against `https://www.traicaybentre.com/` surfaced systemic gaps:

- The WebFetch tool returns LLM-summarised prose, not raw bytes. My
  v1.x DESIGN.md said `#d97706` amber and `Inter` for the brand. The
  actual CSS held `#f97316` orange and `Plus Jakarta Sans` + `Nunito`.
  Every single token was wrong on the first run.
- Closure self-check ran in the host model — by the same model that
  emitted the wrong values. It saw no problem.
- The workflow had no answer for producing a viewable preview. The
  user ran into `<script>` tags hydrating and inline `opacity:0`
  reveal-states leaving content invisible.
- Output dumped at `./DESIGN.md`; running against multiple sites
  collided.

The user's stated goal was sharper than my v1.x targeted: *"the
output will be flawless."* That ruled out "best effort given the
fetch tier" and demanded a deterministic enforcement gate.

## What shipped

**Three new binary subcommands** under `ui designmd`:

| Command | Purpose |
|---|---|
| `extract-tokens <html> --css <css>...` | Grep raw bytes for hex / font-family / custom-property tokens; emit frequency-ranked JSON with line-level provenance. |
| `snapshot <html> --css <css>... --origin <url>` | Strip scripts/preloads, inline CSS, sanitise inline reveal-state styles, absolutise URLs. Emit a self-contained preview HTML. |
| `audit <folder>` | Run 5 audit families (format, source-fidelity, ref-integrity, accessibility, discipline); emit `audit.md` + `audit.json`; exit code drives workflow success. |

**Per-project folder output.** Every successful run writes 8 files
under `./<slug>/`: DESIGN.md, DESIGN.preview.html, source.html,
source.css, tokens.json, run-summary.md, audit.md, audit.json. Slug
derived from URL hostname (`www.` stripped, dots/slashes → dashes);
`--name <slug>` overrides; `--out-dir <path>` relocates.

**Workflow rewrite.** Steps 1, 4, 8, 9 of
`templates/workflows/from-url.md` rewritten:
- Step 1 now requires fetching the linked CSS chunks in addition to
  the HTML. Not optional.
- Step 4 calls `ui designmd extract-tokens` to obtain `tokens.json`;
  the host model draws every token value from this JSON, never from a
  WebFetch prose summary.
- Step 8 writes to the per-project folder and runs `ui designmd
  snapshot` for the preview HTML.
- Step 9 runs `ui designmd audit` and branches on exit code. FAIL
  triggers a 2-attempt re-emit budget before user handoff.

**Knowledge updates.** `knowledge/designmd-format.md` gained a
"Token precision" sidebar that codifies the three-tier source-of-truth
hierarchy: extraction-grade > vision-grade > summary-grade.
`templates/skills/designmd-emit.md` carries a "Source-of-truth
precedence" section pointing at `tokens.json` as canonical.

**Dogfood-validated.** Four URL classes re-run; every emission's
tokens cross-checked against the binary's extraction:

| URL | v1.x guess | v1.1 extraction |
|---|---|---|
| traicaybentre.com | `#d97706` amber + Inter | **`#f97316` + Plus Jakarta Sans + Nunito** |
| stripe.com | `#635bff` violet + system | **`#0d1738` navy + `#533afd` purple + sohne-var** |
| nextjs.org | `#0070f3` + Inter | `#0070f3` (lucky) + **Geist** |
| vercel.com | `#0070f3` (guessed from Next.js similarity) | **`#2c8ce1`** + Geist + **DSEG7 Classic** |

Test count: **627 → 710** (+83 new). All gates green.

## Key decisions

**Audit lives in the binary, not in the host model.** A host-model
self-check is a check by the same agent that produced the output —
v1.x proved that's worth zero. A binary check is bit-identical across
runs, host CLIs, and time. The audit's exit code is the contract; the
workflow cannot bypass it.

**Per-project folder is canonical; single-file dump is gone.** The
v1.x default of `./DESIGN.md` clobbered on multi-URL workspaces and
left users with no audit trail. The new layout puts everything one
run needs in one folder.

**Raw HTML + CSS bytes are the only source of truth for tokens.**
WebFetch's LLM-summarised output is *summary-grade*, useful for
persona prose and component naming but never for hex values, font
names, or dimension values. The audit's source-fidelity family
enforces this — emitted hex absent from `tokens.json` is FAIL, no
appeals.

**Re-emit budget of 2 attempts.** When the audit FAILs, the host
model gets two amendment passes. If the third audit run still FAILs,
the workflow hands off to the user with the full audit report. This
balances "fix small mistakes automatically" against "infinite loop on
intractable inputs."

**WARN exit (2) asks the user.** Not every WARN is a real problem
(some are intentional scope choices like "we don't enumerate every
neutral chrome value"). The workflow surfaces WARN rows and lets the
user accept. The dogfood's three WARN-only sites all landed in this
state by design.

## Risks accepted

- **Regex-only token extraction.** The current extractor handles
  literal hex values, `font-family` declarations, and CSS custom
  properties. It does not parse `calc()`, `color-mix()`, multi-stop
  gradients, or deep `var()` chains. Acceptable for v1.1 — the
  observed brand-tier tokens fall under literal hex 95% of the time.
- **CORS-blocked CSS downgrades to WARN.** If the host CLI can't reach
  a linked stylesheet, the audit emits a WARN row "summary-grade
  only" rather than FAIL. The user is told explicitly which fetch
  tier resolved the run.
- **Single WCAG threshold (4.5:1) for all components.** No per-
  component "this is large text" hint yet. Conservative but readable;
  v1.2 can refine.
- **Preview hot-links images from origin.** The preview HTML inlines
  CSS but doesn't mirror images / fonts. A site takedown breaks the
  preview. Acceptable — the run-folder still carries source.html and
  source.css as a replayable audit trail.

## Next

`/ui:from-url` v1.1 ships feature-complete and audit-gated. The
following are v1.2 candidates, prioritised by expected demand:

1. **DTCG bridge** — `ui designmd export --target dtcg|tailwind` to
   round-trip DESIGN.md ↔ design/*.json. Closes the loop so
   `/ui:generate` can consume the open spec directly.
2. **CSS parser upgrade** — replace regex with a real parser so
   `calc()` / `color-mix()` / deep `var()` chains resolve.
3. **Per-component WCAG hint** — `--text-size: large` on a component
   YAML object so badges fall under 3:1 instead of 4.5:1.
4. **Offline mirror** — `--mirror` flag on `ui designmd snapshot` to
   pull images and fonts into the run folder.
5. **Multi-page crawl** — `ui designmd extract-tokens` taking a list
   of URLs to unify token harvest across an entire site.

None of these is blocking. v1.1 satisfies "the output will be
flawless" as an enforceable contract, not an aspiration.
