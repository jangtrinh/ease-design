## Bottom line

The repository's strongest contribution is process architecture, not its fixed taste rules. It
makes visual direction explicit before code, treats bespoke media and interaction as one coherent
system, and names common convergence failures mechanically. Those mechanisms fit ease-design's
new learning loop because they create inspectable variables and evidence.

## Adopt now

| Mechanism                                             | Why it transfers                                                                | ease-design adoption                                      |
| ----------------------------------------------------- | ------------------------------------------------------------------------------- | --------------------------------------------------------- |
| One reference board per section                       | Prevents a strong hero plus generic remainder; preserves composition variety    | Required evidence for the art-directed benchmark variant  |
| Board-to-code anti-drift analysis                     | Forces implementation to preserve spacing, hierarchy, palette logic, and motifs | Added to art-direction workflow                           |
| Six identity axes                                     | Makes repeated AI defaults observable across builds                             | Stored on every learning-record variant                   |
| Concept-linked signature technique                    | Motion becomes part of the narrative rather than decoration                     | Record one named technique and judge motion craft         |
| Screenshot-safe and reduced-motion-safe initial state | Protects evidence capture and accessibility                                     | Already a Qualified Delivery v2 floor                     |
| Asset coherence and reject separation                 | Prevents mixed visual grades and unused production assets                       | Existing asset provenance; add as a future asset-kit test |

## Test before promotion

- anti-convergence across palette, type, hero, technique, CTA, and shape axes;
- a single “second-read” moment per page;
- per-section composition-anchor variety;
- reference boards generated before code rather than after first implementation;
- signature-technique catalogs selected by concept fit.

These are candidate taste patterns. The learning loop must prove they improve blinded results
across categories before they become defaults.

## Do not adopt

- generated icon sheets: conflicts with the explicit Phosphor interface-icon rule and can harm
  semantic consistency;
- mandatory dark mode: not justified for every audience or product;
- mandatory generated imagery or spectacle on every surface: conflicts with evidence-first,
  performance, and intentional no-image directions;
- provider-specific Higgsfield CLI/model assumptions: `ui` must remain deterministic and
  provider-neutral;
- universal em-dash, serif, palette, or centered-layout bans: useful anti-default prompts in one
  system, but too brittle as cross-product rules;
- banning shared CTA components: conflicts with coherent design systems and maintainable code.

## Main insight

The most valuable idea is separating **visual grammar selection** from **implementation craft**.
Qualified Delivery already controls craft and evidence. Section boards plus identity signatures
give the learning loop a compact representation of visual grammar, making “why this improved”
measurable instead of anecdotal.
